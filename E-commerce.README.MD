🛒 E-Commerce Database Project

📌 Overview

This project is a simple E-commerce database system developed using SQL. It is designed to manage and organize data related to products, customers, orders, payments, and shipping. The system demonstrates how databases are used in real-world online shopping platforms.

---

🗂️ Database

Database Name: "ecommerce"

---

🧱 Tables (Schema Overview)

- Categories – Stores different types of product categories.
- Products – Contains product details such as name, price, stock, and category.
- Customers – Stores customer information like name, contact, and registration details.
- Orders – Maintains records of orders placed by customers.
- Order_item – Stores details of products included in each order.
- Payments – Contains information about payment methods and status.
- Shipping – Tracks shipping and delivery status of orders.

---

🔗 Relationships

- Each product belongs to a category.
- Each order is linked to a customer.
- Order_item acts as a bridge between orders and products.
- Payments are associated with orders.
- Shipping details are linked to orders for delivery tracking.

---

⚙️ Features

- Supports basic database operations like inserting, updating, and deleting data.
- Uses joins to combine data from multiple tables.
- Applies aggregate functions for calculations like totals and averages.
- Includes subqueries for advanced data retrieval.
- Implements window functions for running totals and ranking.
- Uses CASE statements for conditional categorization.
- Handles date and string operations for formatting and analysis.

---

🎯 Conclusion

This project helps in understanding database design, table relationships, and SQL operations used in an E-commerce system. It is useful for academic learning, practical exams, and basic database projects.
